ESP32 Firmware — โฟลเดอร์ esp32/ (ต้องมีบอร์ดจริง)
5.1 ติดตั้ง Arduino IDE และ Board Package
1
เปิด Arduino IDE 2.x → File → Preferences
เพิ่ม Board Manager URL:
https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
2
Tools → Board → Boards Manager → ค้นหา esp32 by Espressif Systems → Install
3
เลือก Board: Tools → Board → ESP32S3 Dev Module
4
ติดตั้ง Library: Sketch → Include Library → Manage Libraries → ค้นหา ArduinoJson v7+ → Install
5.2 ตั้งค่า Credentials
# Master ESP32
cp esp32/config.example.h esp32/config.h
# แก้ไข config.h:
# WIFI_SSID, WIFI_PASSWORD
# FIREBASE_API_KEY, FIREBASE_PROJECT_ID
# MACHINE_EMAIL, MACHINE_PASSWORD
# CF_UPLOAD_URL (URL ของ Cloud Function)

# Slave ESP32
cp esp32/config_slave.example.h esp32/config_slave.h
# แก้ไขเหมือนกัน (ใช้ account แยกกัน)

6
⚡ ทดสอบระบบโดยไม่มี ESP32 Hardware (Simulation Mode)
สามารถทดสอบ Web App + AI ได้ครบวงจรโดยไม่ต้องมีบอร์ด ESP32 — ใช้วิธีต่อไปนี้จำลองการทำงานของเครื่อง
วิธีที่ 1 — แก้ Firestore ด้วยมือ (Firebase Console)
1
เปิด Firebase Console → Firestore → machines → Gloop_01
2
สมัครสมาชิก/Login บน Web App (localhost:3000/login) → กด เริ่มใช้เครื่อง → status จะเปลี่ยนเป็น READY
3
ใน Firestore Console แก้ field status เป็น PROCESSING และ session_score เพิ่มขึ้น → Web App จะแสดงผลทันที (real-time listener)
วิธีที่ 2 — ทดสอบ AI โดยอัปโหลดภาพผ่าน curl / Python
# ตรวจสอบว่า AI service รันอยู่ (python api.py)
# ดูสถานะ service
curl http://localhost:8000/health
curl http://localhost:8000/status

# จำลองขวดถูกตรวจ: เขียน status = "ready" ใน Firestore ด้วยมือ
# แล้วใส่ label_storage_path ชี้ไปที่ภาพขวดใน Firebase Storage
# AI listener จะดึงภาพมา inference และเขียนผลกลับอัตโนมัติ
วิธีที่ 3 — ใช้ Firebase Emulator (Local ทั้งหมด ไม่ต้อง Internet)
# ติดตั้ง emulator suite
firebase emulators:start --only firestore,auth,storage,functions

# เปลี่ยน web/.env.local ให้ชี้ emulator
NEXT_PUBLIC_USE_EMULATOR=true   # (ถ้า app รองรับ)

# รัน web app ตามปกติ — จะเชื่อมกับ emulator แทน Firebase จริง
